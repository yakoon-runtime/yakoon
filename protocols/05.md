### Thema: loging an Plattform - und/nicht an Domain
Nachdem wir uns an der Plattform angemeldet haben, ist es manchmal notwendig, sich erneut anzumelden. Manchmal 
aber auch nicht. Das kann jede Domain selbst entscheiden. Dazu kann die Domain einen entsprechenden Text ausgeben, wenn das notwendig ist. Der Ort dafür könnte das Command switch sein. Switch gibt dann eine Willkommensmeldung aus und informiert, wie die Anmeldung erfolgen soll. Das Command Switch schaltet die domain dann in der Session auf die aktuelle domain. Dazu braucht es noch keinen Account.

**Problem**
- session.command_groups = account.groups wird durch den aktuellen Account überschrieben. command_groups
  wird an den verschiedensten Stellen geprüft. Diese werden bei jeder Session beim initialisieren durch
  die domains festgelegt. Somit legt jede Domain fest, was eine Session per Default tun darf 
  (Festlegung der commandsets.) Durch die Anmeldung (Account) kommen nun aber weitere groups hinzu. 
  Diese müssen der command_groups hinzugefügt werden, um nicht an allen Stellen im Code immer prüfen zu müssen:
  session.commandgroups + account. 

  Überlegung: Nachdem jede Session bei jedem Request neu aufgebaut, und der Account geladen wird (wenn account_id gesetzt), können wir die 
  account.cmd_groups direkt in die session übergeben und müssen diese nicht seperat an der Session verwalten.

  - Bedeutet: Nur der Account speichert die cmd_groups später im Storage. Die Session bekommt ihre gruppen druch die Domains + den Account 
    batch: login stefan; switch mud; help


### Thema: command_groups
command_groups beinhaltet die Bezeichnung aller gültigen commandsets. Diese werden von den Domains bereitgestellt und in die Session initial übertragen. 

**Probleme**
- Namen könnten doppelt sein -> Lösung: Prefix in Namen angeben!
